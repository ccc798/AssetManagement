# 多语言支持设计

**目标：** 中英文双语界面，通过 Riverpod 驱动，用户可在设置中选择语言

---

## 架构

```
lib/core/i18n/
  ├── translations.dart    # 翻译映射表（key → [zh, en]）
  └── locale_provider.dart # 语言选择 provider + t() 工具函数

所有页面通过 t('key') 获取当前语言的文本，替换硬编码字符串。
```

---

## 语言策略

| 用户选择 | 行为 |
|----------|------|
| **默认（跟随系统）** | 系统语言为 zh/zh-CN/zh-TW/zh-HK → 中文；其他所有语言 → 英文 |
| **中文** | 强制中文 |
| **English** | 强制英文 |

---

## 翻译映射表

```dart
class Translations {
  static const Map<String, Map<String, String>> _strings = {
    'app.name':         {'zh': '资产管理',          'en': 'Asset Management'},
    'home.title':       {'zh': '资产管家',          'en': 'Home'},
    'add.title':        {'zh': '添加物品',          'en': 'Add Item'},
    'edit.title':       {'zh': '编辑物品',          'en': 'Edit Item'},
    // ... 约 200-300 条
  };

  static String t(String key, String locale) {
    return _strings[key]?[locale] ?? _strings[key]?['en'] ?? key;
  }
}
```

---

## Provider

```dart
final localeProvider = StateProvider<Locale>((ref) {
  // 默认跟随系统：通过平台检测
  return Locale('zh');
});
```

## 设置页

在「主题设置」下方添加「语言设置」：

```
🌐 语言
  ○ 跟随系统（默认）
  ○ 中文
  ○ English
```

## 工作方式

1. `translations.dart` 定义 `t(key)` 函数
2. 所有页面调用 `t('key')` 替代硬编码中文
3. `localeProvider` 监听用户选择 + 系统语言
4. `app.dart` 读取 localeProvider 设置 `MaterialApp.locale`
5. 设置页添加语言选择入口
